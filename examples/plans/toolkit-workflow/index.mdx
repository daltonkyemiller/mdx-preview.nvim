# MDX site toolkit

This is a checked-in plan, not a chat-only artifact. It uses ordinary Markdown for the narrative and components only where the structure benefits from a visual treatment.

<Callout title="Outcome">
  An agent or teammate can author this file, preview it locally, switch the viewer theme, export one HTML file, or build
  a static directory without creating a separate app.
</Callout>

## Try it

<CodeBlock language="sh" title="Preview and export">
  pnpm mdx-preview serve examples/plans/toolkit-workflow --open pnpm mdx-preview build examples/plans/toolkit-workflow
  --out dist/toolkit-workflow pnpm mdx-preview build examples/plans/toolkit-workflow --out dist/toolkit-workflow-html
  --single-file
</CodeBlock>

Use the toolbar at the top of the viewer to test the theme controls and download a standalone HTML export. The exported file keeps the System color preference behavior; a normal static build is the better choice for a published site.
